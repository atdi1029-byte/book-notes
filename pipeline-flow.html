Book Project Pipeline

How Claude processes a book from PDF to published HTML

Setup & Ship
Processing
Writing
Decision
Setup
Scaffold Book
Create folder, extract cover from PDF, generate template files (summary.md, notes.md, index.html)
./build.sh scaffold "Title" "Author" ...
1
Processing
Run bookai (20-page batches)
Split PDF into batches. Launch parallel Claude invocations. Try Sonnet first, fall back to Haiku on content filter errors. Write to disk after EACH batch.
./bookai "book.pdf" "Folder" 1 END 20 5 3
  • 3 parallel waves at a time
  • Auto-retry with 30s delay (max 2)
  • Track pages in .pages_done
2
Gap-Fill Pass
Check .pages_done against expected range. Re-run any failed/missing pages individually. A book is NOT complete until every page is processed.
automatic (built into bookai)
3
Dedup + Verify
Remove duplicate chapter sections (keep longest). Report chapter coverage, page gaps, final line count.
Python scripts inside bookai
4
Raw notes > 3000 lines?
YES → Compact to ~1500-2000 lines NO → Keep as-is
Compact Notes
Back up raw notes as notes.md.raw. Use 2 parallel agents (first half + second half), merge results. Remove redundant examples, keep core arguments + best evidence.
notes.md.raw → notes.md (~1500-2000 lines)
5
Writing
Write summary.md
One-line thesis, 8-12 key takeaways, thematic breakdowns, contrarian ideas, and author proposals.
~200-500 lines
6
Write index.html (DIRECTLY)
Read notes deeply, then CLOSE them. Write from understanding — not paragraph by paragraph. Original prose, not reformatted notes. Never use subagents for this step.
1200-1500 lines, ~70 char wrap
  • Chapters first (5-8 paragraphs each)
  • Key Takeaways, Contrarian Ideas
  • Notable Quotes (8-15 blockquotes)
  • Allegories & Cultural References
  • Proposals/Frameworks
  • Referenced Works
  • About the Author + Why It Matters
  • Bold 40-70 key terms
7
HTML Checklist passes?
YES → Ship it NO → Fix (lines, bolds, quotes...)
Ship
Add to Shelf
Add book card HTML to main index.html under the appropriate category shelf.
./build.sh add-shelf "Title" ...
8
Git Push
Commit all files and push to GitHub Pages. Book is now live and syncs bookmarks via Apps Script backend.
git push (SSH port 443)
  • book.js handles bookmark sync
  • Progress bar tracks reading
  • JSONP to Google Sheets backend
